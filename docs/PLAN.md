# 工事写真整理・写真帳作成 Webアプリ 実装プラン

**作成日**: 2026-07-06
**作業ブランチ**: `branch5`（完成後 main へマージ）

---

## 1. 背景（なぜ作り直すのか）

### 現状の道具と痛点

| 道具 | 役割 | 問題 |
|---|---|---|
| 本リポジトリの PySide6 アプリ / web/index.html | 写真の選別・リネーム | 実務者の作業フローがUIに反映されていない。Web版はスクリプト生成方式で回りくどい。Vercel非対応 |
| 工事写真帳マルチVer16.xlsm（実務の本体） | 写真帳の作成・印刷 | 完成度は高いが、**取込がファイル名順**のため、先に写真を撮影/選別順にリネームしておく前処理が全て手作業 |

### 実務フロー（アップロード資料・実データ「佐波川」から確定）

1. 現場で撮影：弁栓調査（全景・接写・内部・測定）／全景調査／メーター調査（全景・近景・接写・内部）
2. 写真整理：撮影箇所ごとに番号を振り分けてリネーム ← **ここが一番つらい手作業**
3. Excelマクロで写真帳作成（3枚/4枚タイプ、余白コマ、コマ番号、メモ欄、撮影日）
4. 位置図作成

### 命名規則（実データ book2.xlsm の貼付写真名から確定）

| 調査種別 | 形式 | 実例 | 構造 |
|---|---|---|---|
| 弁栓 | `V{箇所}-{ラベル}` | V1-1〜V1-4, V2-1, V2-2, … V5-4 | 箇所番号 × ラベル番号（1=全景, 2=接写, 3=内部, 4=測定）。箇所ごとに枚数可変 |
| 全景 | `P{連番}` | P01〜P22 | 単純連番（ゼロ埋め2桁） |
| その他資料 | `S{群}-{連番}` | S1-1〜S1-7, S2-1, S3 | 群 × 連番 |
| メーター | `M{箇所}-{ラベル}`（推定） | — | ラベル: 1=全景, 2=近景, 3=接写, 4=内部 |

メモ欄（Excel C列）にはラベル名（全景/接写/内部/測定）が入る。実データではほぼ空欄＝手入力が追いつていない → 自動化対象。

### ゴール

**「LINEで写真を送るときのように、サムネイルをポチポチ選んだ順に、箇所・ラベルが自動で振られてファイル名が一括で変わる」** を核に、最終的には写真帳のExcel出力までWebで完結させる。Vercelにデプロイし、PC・スマホ両対応。

## 3. 技術選定

| 項目 | 選定 | 理由 |
|---|---|---|
| フレームワーク | **Next.js (App Router) + TypeScript** | Vercelのネイティブ対応。フェーズ2以降の拡張余地 |
| スタイル | Tailwind CSS | モバイルファーストのレスポンシブを素早く |
| 処理方式 | **完全クライアントサイド**（サーバ処理なし） | 写真を外部送信しない（業務データの安全性）。Vercelは静的配信のみ |
| EXIF読取 | `exifr` | 撮影日時ソート。軽量・HEIC対応 |
| 直接リネーム | **File System Access API**（`showDirectoryPicker` → `FileSystemFileHandle.move()` / コピー+削除フォールバック） | PC Chrome/Edge で「ポチポチ→完了」を実現 |
| フォールバック | `<input type="file" webkitdirectory multiple>` ＋ **JSZip** でリネーム済みZIP出力 | スマホ/Safari/Firefox 対応 |
| 状態永続化 | `localStorage`（テンプレート設定）+ IndexedDB（作業セッション） | 誤リロードで選択順を失わない |
| Excel出力(F2) | `exceljs` | 画像埋込・列幅・罫線・セル結合に対応（SheetJS無償版は画像不可） |

**デバイス振り分け**：
- PC Chrome/Edge → フォルダを開いて**その場で直接リネーム**（＋undo用マップをJSON保存）
- スマホ/その他ブラウザ → 同じUIで選別し、**リネーム済みZIPをダウンロード**（自動判定で切替）

## 4. フェーズ1の仕様

### 4.1 画面構成（1画面・モバイルファースト）

```
┌──────────────────────────────────┐
│ [フォルダを開く] [種別: 弁栓▼] [箇所: V3] [設定]│ ← ヘッダ（現在の割当状態を常時表示）
├──────────────────────────────────┤
│ ┌────┐ ┌────┐ ┌────┐            │
│ │写真 │ │写真 │ │写真 │  …サムネイルグリッド │
│ │V1-1 │ │V1-2 │ │    │   （撮影日時順）      │
│ │全景 │ │接写 │ │    │            │
│ └────┘ └────┘ └────┘            │
├──────────────────────────────────┤
│ 次: V1-3 内部  [ラベルスキップ] [次の箇所へ→]   │ ← フッタ（次に振られる番号を予告）
│ [選択順リスト] [プレビュー] [実行]              │
└──────────────────────────────────┘
```

### 4.2 コア操作フロー（LINE風）

1. **フォルダを開く** → 写真をEXIF撮影日時順にサムネイル表示（現場は撮影順に回るため、この並びが正解に近い）
2. **種別テンプレートを選択**：弁栓 / メーター / 全景 / その他 / カスタム
3. **写真をタップ** → その瞬間に「次の番号」（例 `V1-2 接写`）がバッジで貼られる。もう一度タップで解除（以降は自動で詰め直し）
4. サイクル（弁栓=4枚）を撮り切らない箇所は **[次の箇所へ]** でカット（例 V2 は2枚で次へ）。サイクル完了時は自動で次の箇所へ（設定でON/OFF）
5. **プレビュー** → 旧名→新名の一覧・重複や欠番の警告
6. **実行** →
   - PC: フォルダ内で直接リネーム。実行前に `rename_undo_{日時}.json` をフォルダに保存（元名↔新名マップ、xlsmマクロの rename_map.csv と同思想）。Undoボタンで復元
   - スマホ等: リネーム済みコピーをZIPでダウンロード
7. **メモ用データ出力**：`No.` → ラベル名（全景/接写/内部/測定）の対応を **クリップボードコピー（Excel列貼り付け用のタブ区切り）** と **CSV/JSONダウンロード** で出力 → xlsmのメモ欄へ一括貼り付け

### 4.3 種別テンプレート（初期値・設定画面で編集可、localStorage保存）

```ts
type Template = {
  id: string
  name: string          // 弁栓
  prefix: string        // "V"
  mode: "cycle" | "serial" | "group-serial"
  labels: string[]      // ["全景","接写","内部","測定"]  (cycle時)
  separator: string     // "-"
  pad: number           // 連番ゼロ埋め桁数 (P型=2)
}
```

初期テンプレート：
- 弁栓: `V{箇所}-{n}`, cycle=[全景, 接写, 内部, 測定]
- メーター: `M{箇所}-{n}`, cycle=[全景, 近景, 接写, 内部]
- 全景: `P{連番2桁}`, serial
- その他資料: `S{群}-{連番}`, group-serial


## 5. フェーズ2の仕様（写真帳編集 → .xlsx出力）

xlsmマクロの操作体系をWebで再現する。フェーズ1の `assignment`（順序・箇所・ラベル）がそのまま帳票の並び・メモ欄になるため、追加開発は「レイアウト編集UI」と「Excel出力」の2つ。

### 5.1 画面構成（`/album`）

```
┌──────────────────────────────────────┐
│ [シート: 全景|弁栓|その他 +] [タイプ: 3枚▼]      │ ← シートタブ（調査種別ごと）
│ [番号: コマ毎|写真毎] [撮影日: 表示▼] [Excel出力] │
├──────────────────────────────────────┤
│  ページ1                                        │
│  ┌───────────┐  No.1                     │
│  │  写真 V1-1     │  メモ: 全景（自動入力・編集可）│
│  └───────────┘  2026-07-06               │
│  ┌───────────┐  No.2                     │
│  │  写真 V1-2     │  メモ: 接写                 │
│  └───────────┘                           │
│  ┌───────────┐  No.3                     │
│  │ （余白コマ）    │                           │
│  └───────────┘                           │
│  ── ページ2 ──  …                              │
└──────────────────────────────────────┘
```

- **紙面プレビュー型**：A4縦1ページ＝コマN個（3枚/4枚タイプ）の見た目そのままで縦スクロール。印刷結果と同じものを見ながら編集する（マクロのシートをそのまま置き換えるイメージ）
- **フェーズ1からの流し込み**：リネーム確定した写真が選択順に自動配置。メモ欄にはラベル名（全景/接写/内部/測定）が自動で入る（＝資料の要望「Excelに貼り付けた時にメモ欄が自動入力」の実現）
- 複数シート対応：佐波川の実例（全景/弁栓/その他資料）のように、調査種別ごとにシートを分けられる

### 5.2 コマ操作（xlsmマクロの機能対応表）

| xlsmマクロの機能 | Web版での操作 |
|---|---|
| 3枚/4枚タイプ切替（SheetTypeChange） | シートごとのタイプ選択。切替時はコマを再流し込み |
| 余白コマ挿入（AddBlank/DeleteBlank） | コマの「＋余白」ボタンで前に余白挿入／余白の削除。後続は自動でずれる |
| 写真の入替（ExchangePicture/MoveToHere） | コマのドラッグ&ドロップ（スマホは長押し）＋「ここへ移動」 |
| コマ毎に番号振直し（SerialNumbering） | 番号モード「コマ毎」: 余白含め No.1,2,3… |
| 写真毎に番号振直し（PictureNumbering） | 番号モード「写真毎」: 写真のあるコマのみ No. を振り余白は空欄 |
| メモ欄・用語集（UserForm4＋用語集シート） | メモはインライン編集。よく使う語はテンプレートのラベル＋ユーザー辞書（localStorage）から1タップ挿入 |
| 撮影日表示（PictureDate系） | シート設定で ON/OFF・書式（YYYY/MM/DD等）。EXIFから自動 |
| ページ追加（AddPages） | 写真流し込み・余白挿入で自動追加（明示操作不要） |

### 5.3 データモデル

```ts
type Album = {
  sheets: AlbumSheet[]
}
type AlbumSheet = {
  name: string              // "弁栓" など
  layout: 3 | 4             // 1ページのコマ数（将来 2/6/8/12 へ拡張可能な型に）
  numbering: "perFrame" | "perPhoto"   // コマ毎 / 写真毎
  showDate: boolean
  frames: Frame[]           // ページ数はframesから自動計算（= ceil(frames/layout)）
}
type Frame =
  | { kind: "photo"; photoId: string; memo: string; date?: string }
  | { kind: "blank" }
```

`album.ts` は純関数（挿入・削除・移動・番号計算）で実装しユニットテスト対象。写真の実体はフェーズ1の `PhotoItem`（IndexedDB）を参照。

### 5.4 Excel出力（`xlsxExport.ts`）

- **exceljs** で生成。1シート=1ワークシート、A4縦・印刷範囲/余白を設定
- コマ構造は xlsm と同じ「結合セル（写真枠）＋コマ番号行＋メモ欄」を再現：
  - 写真: 結合セル位置に `addImage`（アスペクト比維持で枠内フィット、長辺縮小で容量削減）
  - コマ番号: `No.{n}`（番号モードに従う）
  - メモ欄: ラベル名＋自由記入の内容
  - 撮影日: 設定に応じて表示
- 出力は素の `.xlsx`（マクロなし）。既存xlsmの「マクロ無し保存(SaveWOMacro)」相当の成果物に直行するイメージ
- 生成もクライアントサイドで完結（写真を外部送信しない方針を維持）

### 5.5 フェーズ2 実装ステップ

1. `lib/album.ts` データモデル＋操作関数＋ユニットテスト（挿入/削除/移動/番号2モード）
2. フェーズ1完了データ → Album への流し込み（シート分割・メモ自動入力）
3. `/album` 紙面プレビューUI（3枚/4枚、ページ区切り表示）
4. コマ操作（D&D入替、余白挿入/削除、メモ編集、番号モード切替、撮影日ON/OFF）
5. `xlsxExport.ts` exceljs出力（画像埋込・結合セル・罫線・印刷設定）
6. スマホ実機調整（長押しD&D、プレビューの拡大縮小）
7. セッション永続化（IndexedDB）で「現場で選別→帰社後に帳票仕上げ」を跨げるように

## 6. 検証方法

### フェーズ1
1. **ユニットテスト**: `naming.ts` — 弁栓5箇所（枚数可変 4,2,4,4,4）→ V1-1〜V5-4 が実データと一致すること、P型ゼロ埋め、途中解除時の詰め直し
2. **ローカルE2E**: サンプル写真フォルダ（EXIF日時入り）で `npm run dev` → 選択→リネーム実行 → フォルダ内のファイル名が想定通りか、undo JSONで復元できるか
3. **フォールバック**: Chrome DevToolsのデバイスモード＋Safariで ZIP モードに自動切替されること、ZIP内のファイル名が正しいこと
4. **実務検証（ユーザー実施）**: 実際の調査写真1現場分をリネーム → 工事写真帳マルチVer16.xlsm の一括取込に流し、狙った順・コマにハマること、メモTSVをメモ欄に貼り付けられること

### フェーズ2
5. **ユニットテスト**: `album.ts` — 余白挿入で後続がずれること、番号2モード（コマ毎/写真毎）が xlsm の SerialNumbering / PictureNumbering と同じ結果になること
6. **出力検証**: 生成した .xlsx を実際のExcelで開き、佐波川の実物（3枚タイプ）と見比べて写真位置・No.・メモ・撮影日・印刷プレビューが同等であること
7. **一気通貫**: スマホで写真選別 → PCで帳票仕上げ → .xlsx出力 → 印刷、まで通しで実施

### 共通
8. **Vercel**: デプロイ後、会社PC（Chrome/Edge）とスマホの両方で動作確認
